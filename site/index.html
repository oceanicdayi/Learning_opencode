RESEARCH PIPELINE

從資料到網站的可重現流程

01

資料產生／匯入

以固定亂數種子建立教學資料；真實研究時替換為正式 catalog。

02

科學計算

Python 模組執行 MAXC、規模頻度分布與每月 3 個月移動窗。

03

自動測試

針對 bin、tie handling 與時間窗建立 unittest，避免方法被改壞。

04

資料發布

分析結果輸出為 JSON，前端只負責視覺化，不在瀏覽器內重算科學結果。

05

GitHub Pages

每次 main 更新後由 Actions 重建資料、測試並部署網站。

OPENCODE WORKFLOW

可直接使用的任務提示

地震資料品質檢查

閱讀 AGENTS.md。先檢查 catalog 的時間、經緯度、深度、規模、缺值與重複事件。先報告問題與預計修改檔案，不要直接改資料。

科學論文 Methods

只根據 src、configs 與 outputs 撰寫 Methods。不得補寫不存在的步驟；未驗證引用標成 [AUTHOR CHECK]。

Reviewer 式審查

以 SRL reviewer 角度檢查資料洩漏、空間偏差、時間窗可比性、樣本數、方法限制與過度宣稱。只提出可執行問題，不修改正文。

建立新功能

先閱讀測試與資料 schema，提出計畫及風險。完成後執行測試，列出修改檔案、命令、結果與尚未解決限制。
