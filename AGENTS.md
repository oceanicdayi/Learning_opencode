# OpenCode project rules

## 目的

本專案用於地震資料處理、可重現研究、科學論文撰寫、監測系統設計與互動網站部署。

## 科學誠信

- 不得捏造觀測值、模型結果、引用、事件或測站資訊。
- 明確區分觀測資料、計算結果、假設、解釋與未來工作。
- 不得直接修改 `data/raw/`；所有衍生資料寫入 `data/processed/` 或 `site/data/`。
- 論文中的數字、圖與表必須能追溯到程式、設定與輸入資料。
- 未驗證引用必須標成 `[AUTHOR CHECK]`，不得憑記憶建立 bibliography。

## 工作方式

修改前：
1. 閱讀 README、AGENTS、相關設定與測試。
2. 先報告問題、假設、預計修改的檔案及原因。
3. 優先採用小型、可審查、可回復的變更。

修改後：
1. 執行測試與資料重建。
2. 回報實際修改檔案、執行命令、測試結果與限制。
3. 未經明確要求不得 push、部署、刪除資料或改寫 Git 歷史。

## 地震資料規則

- 保存時間格式、時區、單位、座標系統與缺值定義。
- 事件資料切分以 event level 為原則，避免 station sample 洩漏。
- Mc 分析必須回報方法、bin width、視窗、樣本數與品質標記。
- MAXC 僅作為教學或快速估計；正式研究需比較其他 completeness 方法與不確定性。

## 論文規則

- Methods 必須與實際程式一致。
- Results 與 Discussion 分離。
- Reviewer agent 應檢查資料洩漏、選樣偏差、空間偏差、時間窗可比性、過度宣稱與不適當指標。

## Definition of done

- 從乾淨環境可執行。
- 測試通過。
- 輸出位置明確且可重現。
- 假設與限制已記錄。
- README 或 manuscript 文件已同步更新。
